1. 问题形式化

模式识别问题定义

给定传感器时间序列 S = {s₁, s₂, ..., sT}，其中每个采样点 st ∈ ℝ⁹ (3轴加速度 + 3轴陀螺仪 + 3轴磁力计)，目标是学习一个判别函数：

f: ℝ128×9 → 𝒴 = {0,1,2,3,4,5}

将每个 128 帧滑动窗口映射到 6 类活动标签。

类别定义与混淆风险预判

类别标签核心特征易混淆类
静坐0静止 + 大腿水平倾斜站立
站立1静止 + 躯干垂直静坐
走路2周期性加速度 + 低角速度上楼、下楼
上楼3垂直加速度 + 步频走路、下楼
下楼4垂直加速度(反向) + 冲击走路、上楼
跑步5高幅值 + 高频 + 大角速度—

💡 预判走路/上楼/下楼是核心难点——仅靠加速度几乎不可分，需多传感器融合

3. 传感器标定 — 误差来源与建模

⚠️ 未标定的 MEMS 原始数据带有系统误差，会让后续所有特征「失真」——这是数据治理的第一道关

加速度计误差

误差模型 (12参数):

acal = M⁻¹(araw − b)

其中 M 为 3×3 刻度因子+非正交矩阵，b 为 3×1 零偏向量

误差类型:

  • 零偏 (bias) — 静止时读数≠0
  • 刻度因子 — 各轴灵敏度不一致
  • 非正交 — 三轴不完全垂直

约束条件:

静止时合加速度模长 = 1g

陀螺仪误差

主要误差:

  • 零偏 (bias) — 静止时角速度≠0
  • 零偏不稳定性 — 零偏随时间缓慢漂移
  • 角度随机游走 (ARW) — 高斯白噪声积分

标定方法:

  • 静止N秒 → 取角速度均值 = 零偏
  • 运行时直接减去零偏
  • （进阶）Allan方差分析噪声特性

磁力计误差

误差模型:

  • 硬铁 (Hard Iron) — 传感器周围永磁体产生的固定偏置 → 椭球平移
  • 软铁 (Soft Iron) — 铁磁材料对地磁场的扭曲 → 椭球形变

数学模型:

mcal = S⁻¹(mraw − h)

h: 硬铁偏置, S: 软铁矩阵

💡 标定本质是最小二乘参数估计问题 — 与线性回归/线性判别共享数学基础

5. 数据采集协议与标注策略

采集协议设计

传感器佩戴腰部绑带固定 (记录位置，禁止中途更换)
采样率100 Hz (依 Nyquist 定理: 人体运动 ≤20Hz)
被试方案自采1人 + 朋友1人 + UCI HAR 30人 = 32人
每次时长30 秒 (3000 样本点)
采集环境室内平地 + 真实楼梯

标注协议

标注方式动作前标注 — 采集前输入动作名，全程标记为同一标签
过渡处理倒计时 3-2-1 后开始，避免过渡段混入
一致性保证30秒全程同一动作，中间不切换

数据划分策略 (防泄漏)

训练/测试划分分层随机 80/20 (stratify=标签)
交叉验证5-fold — 每折独立标准化
核心原则⚠️ 同一被试窗口不跨训练/测试集
Pipeline封装sklearn.Pipeline → 标准化只在训练折fit

🚫 数据泄漏 No.1: 标准化在全量数据fit → 必须 Pipeline 只在训练折 fit
🚫 数据泄漏 No.2: 同一被试窗口散落训练和测试 → 必须保证被试独立

预处理流水线

为什么选 128 帧窗口？

  • 128帧 × 10ms = 1.28 秒 — 覆盖完整步态周期 (~1–1.5步)
  • 50%重叠 — 平衡响应速度与计算开销
  • 窗口更新频率: 64帧 × 10ms = 0.64s/次

标准化: z-score

x' = (x − μtrain) / σtrain

μ, σ 仅在训练集上计算 → 测试集/推理时复用

⚡ 防泄漏设计 (sklearn.Pipeline)

StandardScaler.fit() → 仅训练折
StandardScaler.transform() → 测试折 / 推理
严禁: 先标准化再划分数据集!

8. 51维特征体系设计

特征组维度索引计算方法物理含义判别能力
MPU时域统计300–29 每轴: μ, σ, max, min, PTP 运动强度、姿态角度 ★★★ 基础判别
方向感知 ⭐430–33 见下方公式 静止/运动区分、身体姿态 ★★★★★ 坐vs站核心
磁力计时域1534–48 每轴: μ, σ, max, min, PTP 朝向变化、楼层磁场特征 ★★★ 上下楼辅助
磁场幅值249–50 ||mag|| 的 μ, σ 环境磁场稳定性 ★★ 场景判断

⭐ 方向感知特征 (索引 30–33) — 本系统关键创新

acc_mag_mean = (1/N) Σ √(ax²+ay²+az²)

静止≈1g, 运动时波动 → 区分静止/运动

acc_mag_std = σ(||acc||)

加速度幅值波动程度 → 运动剧烈度

tilt_angle = arctan2(√(ax̄²+aȳ²), |az̄|)

传感器相对垂直的倾斜角
→ 坐(大腿水平, 角大) vs 站(躯干垂直, 角小)

gyro_mag_mean = (1/N) Σ √(gx²+gy²+gz²)

角速度幅值 → 静止≈0, 跑步/转身≫0

✅ acc_mag_std 特征重要性排第3, tilt_angle 排第7 — 验证了方向感知特征设计的有效性

10. 评估方法论 — 从"会调包"到"懂算法"

评估协议

方法说明目的
5-fold CV分层5折交叉验证评估泛化能力，每折独立标准化
测试集留出80/20 分层划分最终性能报告
固定随机种子random_state=42确保实验可复现

评估指标体系

指标公式/含义为什么重要
Accuracy(TP+TN)/Total总体正确率
PrecisionTP/(TP+FP)预测为该类的准确率
RecallTP/(TP+FN)该类被查全的能力
F1-score2PR/(P+R)P和R的调和平均
混淆矩阵N×N矩阵定位易混淆类对
CV标准差σ across folds模型稳定性

宏平均 vs 微平均

Macro avg: 每类算指标再平均 → 小类不被大类的指标掩盖
Weighted avg: 按样本数加权 → 反映整体表现
本系统报告 Weighted avg (样本量较均衡: 54~3802)

CV标准差的意义

CV = 0.784 ± 0.039 (95% CI)

标准差 0.039 在不同折之间 ≈ 3.9% 的波动
→ 模型在不同数据划分下表现一致，稳定可靠

标定增益量化

标定前 vs 标定后全链路对比：
Accuracy: 86.2% → 88.4% (+2.2%)
坐/站 F1: 0.89 → 0.93 (+4.5%)
→ 标定对静态姿势识别的增益最为显著

误差模式分析

  1. 走路 ↔ 上楼 (52+87)
    慢走上楼时加速度模式与平地走路高度相似。需磁力计朝向特征 + 气压计绝对高度辅助
  2. 走路 ↔ 下楼 (41+56)
    下楼的垂直冲击特征在窗口内被平均化
  3. 静坐 ↔ 站立 (50+47)
    传感器角度接近时(如瘫坐 vs 倚靠)难以区分

改进路径

  • 短期 引入气压计BMP280 → 绝对高度变化
  • 短期 增加窗口长度消融实验
  • 中期 HMM时序平滑 → 利用活动转移概率
  • 中期 代价敏感学习 → 跌倒检测场景
  • 长期 1D-CNN端到端 → 自动学习最优特征

自写推理引擎 (MicroPython)

不依赖 sklearn — 树结构从 JSON 加载后，逐节点遍历直至叶子，多数投票输出结果。

环节耗时说明

窗口填充1.28s128帧 × 10ms
特征提取~3ms51维, 双层循环+sqrt/atan2
RF推理<5ms15树 × ≤8层 = ≤120次比较
窗口更新0.64s64帧步长 × 10ms
平滑确认~1.3s连续2帧一致

资源占用

Flash (固件+代码+模型)~400KB / 16MB
RAM (特征缓冲+JSON模型)~60KB / 8MB PSRAM

#困难根因分析解决方案效果
1 静止识别为"下楼" 30维时域特征缺少方向感知，传感器倾角信息未被利用 新增4维方向感知特征 (tilt_angle + acc_mag) ✅ 解决 正确识别坐/站
